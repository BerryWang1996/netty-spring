# 1.9.1 Backlog (RC12) Implementation Plan

> **For agentic workers:** Use the **subagent-driven-development** skill to execute this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land 8 of the 9 deferred 1.9.1 backlog items as RC12 — a polish/hardening pass on the cluster path. Defer L1 (proposed fix inadequate; needs a custom Spring Condition class beyond audit scope).

**Architecture:** Three multi-item files (`NatsKvSessionRegistry`, `RedisStreamsReliableBroker`, `ClusterNodeManager`) group their items onto the same subagent to avoid edit conflicts. Five file-disjoint subagents dispatch in parallel; one sequential release task at the end.

**Tech Stack:** Java 17, Spring Boot 2.7.18, Lombok `@Slf4j`, JUnit 5, Mockito, Testcontainers (Redis + NATS).

**Branch:** `feature/1.9.1-backlog-polish` (already created off master at 2bafc26).

**Items in scope (8):** L2, L3, L4, L5, L6, L7, L8, N1. Items deferred again: L1.

**Cross-item risks identified:**
- **L4 + L7 interaction**: L4 chains `removeAllForNode(...).thenRun(deregister)` while L7 awaits scheduler termination on shutdown. If L4's chained callback runs on the registry's own executor (not `reconScheduler`), `awaitTermination(reconScheduler)` won't wait for it — the chained `deregister` can still race shutdown. **Mitigation: L4 MUST use `thenRunAsync(...)` with `reconScheduler` as the explicit executor, AND add a guard `if (state.get() == NodeState.LEFT) return;` at the top of the chained block.**
- **L3 is a behavior tightening** for non-ASCII payloads — release notes must call this out so operators with custom codecs audit their `message-max-size-bytes`.

### L7: Await scheduler termination BEFORE deregister

- [ ] **Step 1: Locate `shutdown()`.** Open `ClusterNodeManager.java`, find `shutdown()` (~line 280-320). Note the current sequence: cancel futures → `heartbeat.deregister(nodeId)` → schedulers `.shutdown()`.

- [ ] **Step 2: Reorder + add `awaitTermination`.** Move scheduler shutdown + awaitTermination BEFORE the deregister call:

```java
// Reordered shutdown sequence:
// 1. Cancel scheduled futures (signal: no more new tasks)
if (heartbeatFuture != null) heartbeatFuture.cancel(false);
if (reconciliationFuture != null) reconciliationFuture.cancel(false);
if (resyncFuture != null) resyncFuture.cancel(false);

// 2. Initiate scheduler shutdown
heartbeatScheduler.shutdown();
reconScheduler.shutdown();

// 3. Bounded wait for in-flight tasks to complete (mirrors CoalescingRegistryWriter.shutdown)
try {
    if (!heartbeatScheduler.awaitTermination(5, java.util.concurrent.TimeUnit.SECONDS)) {
        log.warn("heartbeatScheduler did not terminate within 5s — forcing shutdownNow");
        heartbeatScheduler.shutdownNow();
    }
    if (!reconScheduler.awaitTermination(5, java.util.concurrent.TimeUnit.SECONDS)) {
        log.warn("reconScheduler did not terminate within 5s — forcing shutdownNow");
        reconScheduler.shutdownNow();
    }
} catch (InterruptedException ie) {
    Thread.currentThread().interrupt();
    heartbeatScheduler.shutdownNow();
    reconScheduler.shutdownNow();
}

// 4. Now safe to deregister — no scheduler task can race
try {
    heartbeat.deregister(nodeId);
} catch (Exception ex) {
    log.warn("deregister on shutdown failed for {}", nodeId, ex);
}
```

The existing drain-timeout path (FIX D from RC11) stays — this change is about the cleanup phase that comes after drain.

### L4: Chain dead-node cleanup correctly (RESPECT L7's ordering guarantee)

- [ ] **Step 3: Locate `doReconciliation()` dead-node block.** Open the same file, find the reaper-claim-wins block (~line 478):


- [ ] **Step 4: Chain via `thenRunAsync` on `reconScheduler`.** This is the critical detail — the chained block MUST run on `reconScheduler` so L7's `awaitTermination(reconScheduler)` actually waits for it:

```java
log.warn("Reconciliation detected dead node {} — cleaning up sessions + cache", deadNodeId);
sessionRegistry.removeAllForNode(deadNodeId)
    .thenRunAsync(() -> {
        // Guard against shutdown race: if we transitioned to LEFT, skip the deregister
        if (state.get() == NodeState.LEFT) {
            log.debug("Skipping deregister of dead node {} — local node LEFT", deadNodeId);
            return;
        }
        try {
            heartbeat.deregister(deadNodeId);
        } catch (Exception ex) {
            log.warn("Failed to deregister dead node {}", deadNodeId, ex);
        }
        // Any existing dead-node callbacks fire here (move them inside this block)
    }, reconScheduler)   // <-- explicit executor so L7's awaitTermination covers it
    .exceptionally(ex -> {
        // On cleanup failure, leave node in the nodes-set so next sweep retries
        log.warn("Failed to clean sessions for dead node {} — will retry on next sweep", deadNodeId, ex);
        return null;
    });
```

If `reconScheduler` is rejected because of in-flight L7 shutdown, the `thenRunAsync` will throw a `RejectedExecutionException` — caught by the `.exceptionally(...)` and logged. Safe.
